模块化

在 es6 未发布之前，JavaScript 没有语言层面上的模板解决方案，CommonJS 和 AMD 分别是服务端的解决方案和浏览器的解决方案。

es6 module 的设计是静态的，使得其编译的时候就能够确定各个模板的依赖关系。CommonJS 和 AMD 模块，都只能在运行时确定这些东西。比如，CommonJS 模块就是对象，输入时必须查找对象属性。

箭头函数

注意

  • 箭头函数没有自己的 this 对象
    • 箭头函数内部 this 的绑定在函数生成时的外部的 this
    • 普通函数内部的 this 指向函数运行时所在的对象
  • 不可以当作构造函数，也就是说，不可以对箭头函数使用 new 命令，否则会抛出一个错误
  • 不可以使用 arguments 对象，该对象在函数体内不存在。如果要用，可以用 rest 参数代替
  • 不可以使用 yield 命令，因此箭头函数不能用作 Generator 函数

ES7 新特性

  • 数组 includes()方法，用来判断一个数组是否包含一个指定的值，根据情况，如果包含则返回 true，否则返回 false。
  • a ** b 指数运算符，它与 Math.pow(a, b)相同。

incldes() indexOf() 区别

  • 相同
    • 都可以用来判断数组中是否存在某个值
  • 不同点
    • includes 返回的是 boolean 类型，indexOf 返回的是数组下标，-1 代表不存在。
    • 当数组存在 NAN 时，indexOf 无法判断，includes 可以判断
    • 当数组存在空值时，includes 会认为空值是 undefined，indexOf 不会。

正则表达式反向断言

(?<=y)x

匹配’x’仅当’x’前面是’y’.这种叫做后行断言。

例如，/(?<=Jack)Sprat/会匹配到’ Sprat ‘仅仅当它前面是’ Jack ‘。/(?<=Jack|Tom)Sprat/匹配‘ Sprat ’仅仅当它前面是’Jack’或者是‘Tom’.但是‘Jack’和‘Tom’都不是匹配结果的一部分。

(?<!y)x

仅仅当’x’前面不是’y’时匹配’x’，这被称为反向否定查找。

例如, 仅仅当这个数字前面没有负号的时候，/(?<!-)\d+/ 匹配一个数字。
/(?<!-)\d+/.exec(‘3’) 匹配到 “3”.
/(?<!-)\d+/.exec(‘-3’) 因为这个数字前有负号，所以没有匹配到。

Promise.any与Promise.race十分容易混淆，务必注意区分，Promise.race 一旦某个promise触发了resolve或者reject，就直接返回了该状态结果，并不在乎其成功或者失败
