卷技术

将容器目录与本地目录进行绑定挂载，使容器数据进行持久化以及方便容器内数据修改

具名和匿名挂载

所有docker的卷在未指定目录下，都是在/var/lib/docker/volumes/xxxxx/_data的目录下，匿名的情况xxxx是随机生成的字符串，通常使用具名的方式进行挂载

DockerFile

dockerfile是用来构建docker镜像的脚本文件，命令参数脚本

  1. 编写dockerfile文件
  2. docker build 生成镜像
  3. docker run 生成容器
  4. docker push 发布镜像（docker hub、阿里云镜像仓库）

    查看官方文档的dockerfile

这些官方镜像的命令行缺少很多，若有需要可以自行添加

DockerFile构建过程

基础知识：

  1. 每个关键字指令必须大写
  2. 执行从上到下顺序执行
  3. #表示注释
  4. 每一个命令都将创建一个新的镜像层，并提交

dockerfile是面向开发的，要发布项目，制作镜像就需要编写dockerfile

Docker 网络

docker 内部容器通信以及容器与外部通信是运用到一个桥接的技术。容器内部通信相当于内网通信技术。只是连接的方法是evth-pair技术。

容器内网络互联 –link（不建议）

本质上是在tomcat02上修改了host，指定了tomcat01的ip

容器互联-自定义网络

查看所有网络

说明：

  • bridge ：桥接模式 docker默认（自己创建网络也是用bridge）
  • none：不配置网络
  • host：和宿主机共享网络

自定义网络的好处
