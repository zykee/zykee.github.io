设计模式学习-观察者模式

观察者模式是一种很经典的设计模式，允许你定义一种订阅机制， 可在对象事件发生时通知多个 “观察” 该对象的其他对象。可以良好的使代码的解耦。

应用场景

  • 当一个对象状态的改变需要改变其他对象，或实际对象是事先未知的或动态变化的时，可使用观察者模式。

比如按钮的点击事件，当用户点击该按钮时，就会触发该点击事件。

  • 当应用中的一些对象必须观察其他对象时，可使用该模式。但仅能在有限时间内或特定情况下使用。

订阅列表是动态的， 因此订阅者可随时加入或离开该列表。

代码实现

js是更偏向于函数式编程，对于 OO 的设计模式实现会有所不同。

// 发布者
class Publisher {
  constructor() {
    this.listeners = new Map();
  }

  subscribe(key, event) {
    if (this.listeners.has(key)) {
      this.listeners.set(key, this.listeners.get(key).push(event));
    } else {
      this.listeners.set(key, [event]);
    }
  }

  unSubscribe(key, event) {
    if (this.listeners.has(key)) {
      const events = this.listeners.get(key);
      const index = events.indexOf(event);
      if (index >= 0) {
        events.splice(index, 1);
        this.listeners.set(key, events);
      }
    }
  }

  notifyFunc(key, ...args) {
    if (this.listeners.has(key)) {
      this.listeners.get(key).map((listener) => listener.apply(null, args));
    }
  }

  notifyClass(key, ...args) {
    if (this.listeners.has(key)) {
      this.listeners.get(key).map((listener) => listener.update(...args));
    }
  }
}
// 类 订阅者
class Subscriber {
  update(...args) {
    console.log("klass", args);
  }
}
// 函数
let subscriber = (...args) => {
  console.log("func", args);
};

const pub = new Publisher();
const sub = new Subscriber();
pub.subscribe("func", subscriber);
pub.subscribe("klass", sub);

pub.notifyFunc("func", 1, 2, 3);
pub.notifyClass("klass", 1, 2, 3);
console.log("---");
pub.unSubscribe("func", subscriber);
// pub.unSubscribe("klass", sub);

pub.notifyFunc("func", 1, 2, 3);
pub.notifyClass("klass", 1, 2, 3);

优缺点

优点

  • 开闭原则。无需修改发布者代码就能引入新的订阅者类 （如果是发布者接口则可轻松引入发布者类）。
  • 可以在运行时建立对象之间的联系。

缺点

  • 订阅者的顺序是随机的（本文代码实现是先进先触发）
